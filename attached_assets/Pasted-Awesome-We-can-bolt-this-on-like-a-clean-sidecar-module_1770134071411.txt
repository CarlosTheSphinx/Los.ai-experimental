# What you’re building

You want **one unified system** that supports:

1. **Notifications** (system updates: “Deal moved to UW”, “Docs missing”, etc.)
2. **Messages** (two-way conversation tied to a deal or general inbox)
3. **Unread badge counts** on both Admin and User “Messages” tab
4. **Popup/toast** when a new item arrives
5. Works on **Admin side** and **User side** with role-based access
6. Minimal coupling to the rest of the app

To keep it clean:

* **Notifications** are just messages with `type = "notification"` and often `direction = "system"`.
* **Chat messages** are `type = "message"` and `direction = "admin_to_user" | "user_to_admin"`.

---

# 1) Required fields (data model)

You need 3 concepts:

### A) Thread (a “room”)

A thread is either:

* tied to a **deal** (recommended)
* or a general inbox thread

Fields:

* `id`
* `dealId` (nullable)
* `userId` (the partner/user on the thread)
* `createdBy` (admin user id or system)
* `subject` (optional)
* `lastMessageAt`
* `isClosed` (optional)

### B) Message (an event inside a thread)

Fields:

* `id`
* `threadId`
* `senderId` (nullable for system)
* `senderRole` (`admin|user|system`)
* `type` (`message|notification`)
* `body` (text)
* `meta` (JSON: status change, stage, etc.)
* `createdAt`

### C) Read receipts (per user per thread)

Simplest: store `lastReadAt` for each user/thread.

Fields:

* `threadId`
* `userId`
* `lastReadAt`

Unread count is then:

> number of messages created after `lastReadAt` (excluding ones you sent)

# 2) Database schema (SQL migration)

This is Postgres-friendly and also works in many SQL DBs.

```sql
-- threads
CREATE TABLE IF NOT EXISTS message_threads (
  id TEXT PRIMARY KEY,
  deal_id TEXT NULL,
  user_id TEXT NOT NULL,
  created_by TEXT NULL,
  subject TEXT NULL,
  is_closed BOOLEAN NOT NULL DEFAULT FALSE,
  last_message_at TIMESTAMP NOT NULL DEFAULT NOW(),
  created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_threads_user ON message_threads(user_id);
CREATE INDEX IF NOT EXISTS idx_threads_deal ON message_threads(deal_id);

-- messages
CREATE TABLE IF NOT EXISTS messages (
  id TEXT PRIMARY KEY,
  thread_id TEXT NOT NULL REFERENCES message_threads(id) ON DELETE CASCADE,
  sender_id TEXT NULL,
  sender_role TEXT NOT NULL CHECK (sender_role IN ('admin','user','system')),
  type TEXT NOT NULL CHECK (type IN ('message','notification')),
  body TEXT NOT NULL,
  meta JSONB NULL,
  created_at TIMESTAMP NOT NULL DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_messages_thread ON messages(thread_id);
CREATE INDEX IF NOT EXISTS idx_messages_created_at ON messages(created_at);

-- read receipts
CREATE TABLE IF NOT EXISTS message_reads (
  thread_id TEXT NOT NULL REFERENCES message_threads(id) ON DELETE CASCADE,
  user_id TEXT NOT NULL,
  last_read_at TIMESTAMP NOT NULL DEFAULT '1970-01-01',
  PRIMARY KEY (thread_id, user_id)
);

CREATE INDEX IF NOT EXISTS idx_reads_user ON message_reads(user_id);
```

> If your app uses Prisma/Drizzle, you can translate this easily, but SQL is fastest for Replit.

# 3) Express API (isolated router)

Create: `server/routes/messages.js`

This assumes you already have auth middleware that sets:

* `req.user.id`
* `req.user.role` (`admin` / `super_admin` / `user`)

If your roles differ, just map them.

### Endpoints you need

* `GET /api/messages/threads` (list threads)
* `GET /api/messages/threads/:id` (thread + messages)
* `POST /api/messages/threads` (create or get thread by deal/user)
* `POST /api/messages/threads/:id/messages` (send message/notification)
* `POST /api/messages/threads/:id/read` (mark as read)
* `GET /api/messages/unread-count` (badge count)
* `GET /api/messages/stream` (SSE for live popups, optional but recommended)

Here is a ready-to-use router (DB layer is written in a “query(db, …)” style; adjust to your db client):

```js
import express from "express";
import { randomUUID } from "crypto";

export function createMessagesRouter({ db, requireAuth }) {
  const router = express.Router();

  // --- helpers ---
  const isAdminRole = (role) => role === "admin" || role === "super_admin";

  // Guard: admin or user must be logged in
  router.use(requireAuth);

  // List threads user can see
  router.get("/threads", async (req, res) => {
    const userId = req.user.id;
    const role = req.user.role;

    let rows;
    if (isAdminRole(role)) {
      // Admin can see all threads or filter by query params
      const { dealId, userId: filterUserId } = req.query;
      const conditions = [];
      const params = [];
      if (dealId) { params.push(dealId); conditions.push(`deal_id = $${params.length}`); }
      if (filterUserId) { params.push(filterUserId); conditions.push(`user_id = $${params.length}`); }
      const where = conditions.length ? `WHERE ${conditions.join(" AND ")}` : "";
      rows = await db.query(
        `SELECT * FROM message_threads ${where} ORDER BY last_message_at DESC LIMIT 100`,
        params
      );
    } else {
      // User sees only their threads
      rows = await db.query(
        `SELECT * FROM message_threads WHERE user_id = $1 ORDER BY last_message_at DESC LIMIT 100`,
        [userId]
      );
    }

    res.json({ threads: rows.rows ?? rows });
  });

  // Get thread + messages (authorization enforced)
  router.get("/threads/:id", async (req, res) => {
    const threadId = req.params.id;
    const userId = req.user.id;
    const role = req.user.role;

    const threadResult = await db.query(`SELECT * FROM message_threads WHERE id = $1`, [threadId]);
    const thread = (threadResult.rows ?? threadResult)[0];
    if (!thread) return res.status(404).json({ error: "Thread not found" });

    // auth check
    if (!isAdminRole(role) && thread.user_id !== userId) {
      return res.status(403).json({ error: "Not authorized" });
    }

    const messagesResult = await db.query(
      `SELECT * FROM messages WHERE thread_id = $1 ORDER BY created_at ASC LIMIT 500`,
      [threadId]
    );

    res.json({ thread, messages: messagesResult.rows ?? messagesResult });
  });

  // Create/get thread (by dealId + userId)
  router.post("/threads", async (req, res) => {
    const { dealId = null, userId: targetUserId, subject = null } = req.body;
    const requesterRole = req.user.role;

    if (!targetUserId) return res.status(400).json({ error: "userId required" });

    // Users can only create threads for themselves
    if (!isAdminRole(requesterRole) && targetUserId !== req.user.id) {
      return res.status(403).json({ error: "Not authorized" });
    }

    // Reuse existing thread for deal+user if exists
    const existing = await db.query(
      `SELECT * FROM message_threads WHERE user_id = $1 AND deal_id ${dealId ? "= $2" : "IS NULL"} LIMIT 1`,
      dealId ? [targetUserId, dealId] : [targetUserId]
    );
    const existingThread = (existing.rows ?? existing)[0];
    if (existingThread) return res.json({ thread: existingThread });

    const id = randomUUID();
    const createdBy = req.user.id;

    const insert = await db.query(
      `INSERT INTO message_threads (id, deal_id, user_id, created_by, subject)
       VALUES ($1, $2, $3, $4, $5)
       RETURNING *`,
      [id, dealId, targetUserId, createdBy, subject]
    );

    // init read receipts for the user (and optionally for admins if you want)
    await db.query(
      `INSERT INTO message_reads (thread_id, user_id, last_read_at)
       VALUES ($1, $2, '1970-01-01')
       ON CONFLICT (thread_id, user_id) DO NOTHING`,
      [id, targetUserId]
    );

    res.json({ thread: (insert.rows ?? insert)[0] });
  });

  // Send message/notification
  router.post("/threads/:id/messages", async (req, res) => {
    const threadId = req.params.id;
    const { body, type = "message", meta = null } = req.body;

    if (!body || typeof body !== "string") return res.status(400).json({ error: "body required" });
    if (!["message", "notification"].includes(type)) return res.status(400).json({ error: "invalid type" });

    const threadResult = await db.query(`SELECT * FROM message_threads WHERE id = $1`, [threadId]);
    const thread = (threadResult.rows ?? threadResult)[0];
    if (!thread) return res.status(404).json({ error: "Thread not found" });

    const role = req.user.role;
    const userId = req.user.id;

    // auth check: user can only post into own thread; admin can post anywhere
    if (!isAdminRole(role) && thread.user_id !== userId) {
      return res.status(403).json({ error: "Not authorized" });
    }

    const senderRole = isAdminRole(role) ? "admin" : "user";
    const messageId = randomUUID();

    const insert = await db.query(
      `INSERT INTO messages (id, thread_id, sender_id, sender_role, type, body, meta)
       VALUES ($1, $2, $3, $4, $5, $6, $7)
       RETURNING *`,
      [messageId, threadId, userId, senderRole, type, body, meta]
    );

    await db.query(`UPDATE message_threads SET last_message_at = NOW() WHERE id = $1`, [threadId]);

    // NOTE: we do NOT auto-mark read here; the recipient sees it as unread.
    res.json({ message: (insert.rows ?? insert)[0] });
  });

  // Mark read (sets lastReadAt = now)
  router.post("/threads/:id/read", async (req, res) => {
    const threadId = req.params.id;
    const role = req.user.role;
    const userId = req.user.id;

    const threadResult = await db.query(`SELECT * FROM message_threads WHERE id = $1`, [threadId]);
    const thread = (threadResult.rows ?? threadResult)[0];
    if (!thread) return res.status(404).json({ error: "Thread not found" });

    if (!isAdminRole(role) && thread.user_id !== userId) {
      return res.status(403).json({ error: "Not authorized" });
    }

    // For admins: you can store reads too (optional). Here we store reads for anyone.
    await db.query(
      `INSERT INTO message_reads (thread_id, user_id, last_read_at)
       VALUES ($1, $2, NOW())
       ON CONFLICT (thread_id, user_id)
       DO UPDATE SET last_read_at = NOW()`,
      [threadId, userId]
    );

    res.json({ ok: true });
  });

  // Unread count (for badge)
  router.get("/unread-count", async (req, res) => {
    const role = req.user.role;
    const userId = req.user.id;

    let count = 0;

    if (isAdminRole(role)) {
      // Admin unread = messages from users after admin last_read_at
      // If you want per-admin read tracking, this works as-is.
      const q = await db.query(
        `
        SELECT COUNT(*)::int AS count
        FROM messages m
        JOIN message_threads t ON t.id = m.thread_id
        LEFT JOIN message_reads r ON r.thread_id = t.id AND r.user_id = $1
        WHERE m.sender_role = 'user'
          AND m.created_at > COALESCE(r.last_read_at, '1970-01-01')
        `,
        [userId]
      );
      count = (q.rows ?? q)[0]?.count ?? 0;
    } else {
      // User unread = messages from admins/system after user last_read_at
      const q = await db.query(
        `
        SELECT COUNT(*)::int AS count
        FROM messages m
        JOIN message_threads t ON t.id = m.thread_id
        LEFT JOIN message_reads r ON r.thread_id = t.id AND r.user_id = $1
        WHERE t.user_id = $1
          AND m.sender_role IN ('admin','system')
          AND m.created_at > COALESCE(r.last_read_at, '1970-01-01')
        `,
        [userId]
      );
      count = (q.rows ?? q)[0]?.count ?? 0;
    }

    res.json({ unreadCount: count });
  });

  return router;
}
```

## A) API helper

Create `client/src/lib/messagesApi.ts`:

```ts
export async function getUnreadCount() {
  const res = await fetch("/api/messages/unread-count", { credentials: "include" });
  if (!res.ok) throw new Error("Failed unread count");
  return res.json() as Promise<{ unreadCount: number }>;
}

export async function listThreads() {
  const res = await fetch("/api/messages/threads", { credentials: "include" });
  if (!res.ok) throw new Error("Failed threads");
  return res.json() as Promise<{ threads: any[] }>;
}

export async function getThread(id: string) {
  const res = await fetch(`/api/messages/threads/${id}`, { credentials: "include" });
  if (!res.ok) throw new Error("Failed thread");
  return res.json() as Promise<{ thread: any; messages: any[] }>;
}

export async function sendMessage(threadId: string, body: string, type: "message"|"notification"="message") {
  const res = await fetch(`/api/messages/threads/${threadId}/messages`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    credentials: "include",
    body: JSON.stringify({ body, type })
  });
  if (!res.ok) throw new Error("Failed send");
  return res.json();
}

export async function markRead(threadId: string) {
  const res = await fetch(`/api/messages/threads/${threadId}/read`, {
    method: "POST",
    credentials: "include"
  });
  if (!res.ok) throw new Error("Failed mark read");
  return res.json();
}
```

## B) Inbox badge component

`client/src/components/InboxBadge.tsx`:

```tsx
import React from "react";
import { getUnreadCount } from "../lib/messagesApi";

export function InboxBadge() {
  const [count, setCount] = React.useState<number>(0);

  React.useEffect(() => {
    let alive = true;

    async function tick() {
      try {
        const data = await getUnreadCount();
        if (alive) setCount(data.unreadCount ?? 0);
      } catch {
        // ignore
      }
    }

    tick();
    const id = setInterval(tick, 15000); // polling every 15s
    return () => { alive = false; clearInterval(id); };
  }, []);

  if (!count) return null;

  return (
    <span style={{
      marginLeft: 8,
      background: "red",
      color: "white",
      borderRadius: 999,
      padding: "2px 8px",
      fontSize: 12,
      lineHeight: "16px"
    }}>
      {count}
    </span>
  );
}
```

> This won’t affect anything else. It just hits one endpoint every 15s.

## C) Messages page (shared)

Create `client/src/pages/MessagesPage.tsx`:

```tsx
import React from "react";
import { listThreads, getThread, sendMessage, markRead } from "../lib/messagesApi";

export default function MessagesPage() {
  const [threads, setThreads] = React.useState<any[]>([]);
  const [activeId, setActiveId] = React.useState<string | null>(null);
  const [active, setActive] = React.useState<{ thread: any; messages: any[] } | null>(null);
  const [draft, setDraft] = React.useState("");

  React.useEffect(() => {
    (async () => {
      const data = await listThreads();
      setThreads(data.threads ?? []);
      if ((data.threads ?? []).length && !activeId) setActiveId(data.threads[0].id);
    })();
  }, []);

  React.useEffect(() => {
    if (!activeId) return;
    (async () => {
      const data = await getThread(activeId);
      setActive(data);
      await markRead(activeId);
    })();
  }, [activeId]);

  async function onSend() {
    if (!activeId || !draft.trim()) return;
    await sendMessage(activeId, draft.trim(), "message");
    setDraft("");
    const data = await getThread(activeId);
    setActive(data);
    await markRead(activeId);
  }

  return (
    <div style={{ display: "flex", height: "calc(100vh - 120px)" }}>
      <div style={{ width: 320, borderRight: "1px solid #ddd", overflow: "auto" }}>
        <div style={{ padding: 12, fontWeight: 700 }}>Inbox</div>
        {threads.map(t => (
          <button
            key={t.id}
            onClick={() => setActiveId(t.id)}
            style={{
              display: "block",
              width: "100%",
              textAlign: "left",
              padding: 12,
              border: "none",
              borderBottom: "1px solid #eee",
              background: t.id === activeId ? "#f5f5f5" : "white",
              cursor: "pointer"
            }}
          >
            <div style={{ fontWeight: 600 }}>{t.subject || (t.deal_id ? `Deal ${t.deal_id}` : "General")}</div>
            <div style={{ fontSize: 12, opacity: 0.7 }}>Last: {new Date(t.last_message_at).toLocaleString()}</div>
          </button>
        ))}
      </div>

      <div style={{ flex: 1, display: "flex", flexDirection: "column" }}>
        <div style={{ padding: 12, borderBottom: "1px solid #ddd", fontWeight: 700 }}>
          {active?.thread?.subject || (active?.thread?.deal_id ? `Deal ${active.thread.deal_id}` : "Messages")}
        </div>

        <div style={{ flex: 1, padding: 12, overflow: "auto" }}>
          {(active?.messages ?? []).map(m => (
            <div key={m.id} style={{ marginBottom: 12 }}>
              <div style={{ fontSize: 12, opacity: 0.6 }}>
                {m.sender_role.toUpperCase()} • {new Date(m.created_at).toLocaleString()} • {m.type}
              </div>
              <div style={{
                padding: 10,
                borderRadius: 10,
                background: m.type === "notification" ? "#fff7e6" : "#f2f2f2",
                border: "1px solid #eee"
              }}>
                {m.body}
              </div>
            </div>
          ))}
        </div>

        <div style={{ display: "flex", gap: 8, padding: 12, borderTop: "1px solid #ddd" }}>
          <input
            value={draft}
            onChange={(e) => setDraft(e.target.value)}
            placeholder="Type a message..."
            style={{ flex: 1, padding: 10 }}
            onKeyDown={(e) => { if (e.key === "Enter") onSend(); }}
          />
          <button onClick={onSend} style={{ padding: "10px 14px" }}>Send</button>
        </div>
      </div>
    </div>
  );
}
```

# 5) Pop-up notifications (simple, safe)

Easiest: when unread count increases, show a toast.

If you already have a toast system, use it. If not, minimal approach: `alert()` is ugly. Better: a tiny toast component.

Logic:

* Poll unread count
* If `newCount > oldCount`, show toast “New message received”

This keeps it isolated and doesn’t require websockets.

# 6) “Deal updates” as notifications (how you’ll use it)

When a deal changes stage, your backend can push a notification:

* Find thread by `dealId + userId` (create if not exists)
* Insert a message with `type="notification"` and `senderRole="system"`

Example payload:

* body: `Deal moved to Underwriting`
* meta: `{ dealId, fromStage, toStage }`

No UI change needed. It will show as a yellow-ish notification bubble.
