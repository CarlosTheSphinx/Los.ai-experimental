Task B: Add Term Sheets “Edit / Void / Resend” lifecycle

Current need:
In my Term Sheets tab (list of documents with statuses: draft/sent/signed/etc):
- I should be able to click “Edit” and return to the full edit screen for that term sheet.
- Even if I already sent a document, I should be able to:
  1) Void/Cancel it
  2) Re-open edit mode
  3) Re-send a new version

Implement as a simple state machine + versioning:

1) Data model changes (choose the simplest approach):
Option 1 (preferred): versioned records
- TermSheet: { id, status, currentVersionId, ... }
- TermSheetVersion: { id, termSheetId, versionNumber, pdfUrl, fieldsJson, createdAt }
- SignatureEnvelope: { id, termSheetVersionId, provider, providerEnvelopeId, status, createdAt }

Option 2: single record + history
- TermSheet has fieldsJson/pdfUrl updated over time
- SignatureEnvelopes reference TermSheet + snapshot fieldsJson at send time

2) Status transitions:
- draft → sent → signed (normal)
- sent → voided (user action “Void”)
- voided → draft (when user clicks “Edit” after voiding, create a new version or reopen draft)
- sent (old envelope) stays voided; resend creates a new envelope tied to the newest version

3) UI behavior in Term Sheets list:
- Actions shown based on status:
  - Draft: Edit, Send, Delete
  - Sent: View, Void, (Edit should first void then open edit), Resend (creates new envelope after void+edit optional)
  - Signed: View (and optionally “Create Amendment” that clones into new draft version)
- Clicking Edit routes to /term-sheets/:id/edit and loads:
  - the latest editable version fields + pdf
- Clicking Void:
  - calls backend: POST /term-sheets/:id/void (or /envelopes/:envelopeId/void)
  - voids in provider (PandaDoc/DocuSign) if integrated
  - updates internal status to voided
- Clicking Resend:
  - creates a new version snapshot if needed
  - creates a new envelope and sets status to sent

4) Back-end endpoints:
- GET /term-sheets/:id (include status + latest version + envelopes)
- POST /term-sheets/:id/versions (clone latest → new draft version)
- POST /term-sheets/:id/send (send latest draft version; validate payload to prevent NaN)
- POST /term-sheets/:id/void (void latest active envelope)
- POST /term-sheets/:id/resend (void active envelope if exists, then send latest version)

Deliverables:
- Implement minimal schema and endpoints
- Update Term Sheets table UI with action buttons and correct routing
- Ensure “Edit” always returns to the full editor screen with fields intact
- Ensure “Void + Edit + Resend” works end-to-end
