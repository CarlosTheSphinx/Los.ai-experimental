Watch for MUI dropdowns opening. Use a MutationObserver on document.body watching for added nodes. When a ul[role="listbox"] appears in the DOM (which happens when any MUI Select dropdown opens):

The visible label: li.innerText.trim().
The option ID by walking the React fiber. Find the property on the <li> DOM element starting with __reactProps$ or __reactFiber$ (property names are minified but follow these prefixes). From the fiber/props, the option's value prop is what we want — that's the ID.


Also identify which field this dropdown belongs to by reading the aria-labelledby attribute on the <ul>. That ID points to a <label> element elsewhere in the DOM; read its innerText to get the human-readable field name (e.g. "FICO Score", "LTV", "Property Type").
Try multiple extraction strategies in order (be defensive — React internals vary):

Strategy A: Read li.__reactProps$<hash>.value directly.
Strategy B: Walk the React fiber: find __reactFiber$<hash> on the <li>, then traverse .memoizedProps.value or .return.memoizedProps.value.
Strategy C: Check if MUI stored the value on a parent element's data-value attribute.
Strategy D (last resort): If all above fail for a given dropdown, log a clear warning like [Lendry] Could not extract IDs for field "FICO" — React fiber structure unexpected.
